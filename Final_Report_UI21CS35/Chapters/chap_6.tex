\chapter{Limitations And Future Work}

\section{Limitations}

Although the \textbf{PrepSmart platform} successfully integrates multiple features for placement preparation, it has certain limitations that can be improved in future versions. 

\vspace{0.2cm}

These limitations highlight areas where the system can be further enhanced to provide a more \textbf{robust, scalable,} and \textbf{user-friendly solution}:

\begin{itemize}[noitemsep, topsep=2pt]
    \item \textbf{Language Support:} Currently, the system supports only \textbf{Python} in the code editor, which restricts users who prefer other programming languages such as Java or C++.
    \item \textbf{Deployment:} The platform operates in a \textbf{local environment} and is not deployed on a cloud server, which limits accessibility, scalability, and real-world usage.
    \item \textbf{AI Evaluation:} The AI Interview module performs \textbf{basic evaluation} based on predefined logic and does not use advanced natural language processing techniques, limiting the depth of feedback.
    \item \textbf{Admin Interface:} At present, administrative operations such as managing users, questions, tests, and content are handled through the \textbf{Django Admin Panel}, which is not directly accessible through the main user interface. This creates a dependency on backend tools instead of a fully integrated system.
    \item \textbf{UI/UX:} The platform lacks refinement in the \textbf{landing page design} and \textbf{user experience}, which can be improved to provide a more engaging first impression.
    \item \textbf{Static Content:} The system relies on \textbf{static question sets,} and \textbf{dynamic} or \textbf{adaptive} question generation is not implemented.
\end{itemize}

\section{Future Work}

The \textbf{PrepSmart platform} has strong potential for future enhancements and expansion. Overall, the following improvements will transform \textbf{PrepSmart} into a more intelligent, scalable, and industry-ready placement preparation ecosystem:

\begin{itemize}[noitemsep, topsep=2pt]
    \item \textbf{Expanded Language Support:} Support for \textbf{multiple programming languages} such as Java, C++, and JavaScript can be added to the code editor to make the platform more versatile.
    \item \textbf{Cloud Deployment:} The system can be deployed on \textbf{cloud platforms} to improve accessibility, scalability, and performance for real-world usage.
    \item \textbf{Integrated Admin Dashboard:} A major improvement would be the development of a \textbf{dedicated admin dashboard within the main application}, allowing administrators to manage users, questions, tests, and analytics directly through the platform.
    \item \textbf{Enhanced UI/UX:} The \textbf{landing page can be enhanced} with improved design, better content presentation, and interactive elements to create a stronger first impression and improve user engagement.
    \item \textbf{Advanced AI Integration:} The AI module can be significantly upgraded using \textbf{advanced Artificial Intelligence} and \textbf{Natural Language Processing (NLP)} techniques to provide deeper evaluation, personalized feedback, and intelligent recommendations.
    \item \textbf{Live Updates:} AI can be integrated to provide \textbf{live hiring updates, company notifications,} and \textbf{job alerts}, helping students stay informed about ongoing placement opportunities.
    \item \textbf{Adaptive Learning:} The system can incorporate \textbf{adaptive learning algorithms} to recommend personalized preparation paths based on user performance.
    \item \textbf{New Modules:} Features such as \textbf{resume analysis, company-specific preparation modules,} and \textbf{mock HR interviews} can further enhance the platform.
\end{itemize}
